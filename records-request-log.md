## Purpose
Track every medical-records request so nothing disappears into phone calls, portals, voicemail, or paperwork delays.

---

## Definition of Done
- every records request is logged
- follow-up dates are assigned
- receipt status is visible
- any fees, forms, or barriers are documented
- received records are stored and marked complete

---

## Operating Principle
Do not rely on memory for records requests. Log the request the same day it happens, including who was contacted and what was requested.

---

## Status Key
- **Planned** = provider identified, request not yet sent
- **Requested** = initial request sent
- **Follow-Up Due** = waiting period passed, needs contact
- **Pending Provider Action** = provider needs release, ID, payment, or form
- **Partial Received** = some records received, file still incomplete
- **Complete** = requested materials received and stored
- **Closed / Not Available** = records unavailable or request no longer needed

## Simple Outreach Script
Use or adapt this when contacting a provider's records office.

> Hello, I am requesting copies of my records for disability and personal documentation purposes. I would like to confirm what records you have, what release or ID you need from me, and the expected turnaround time.
